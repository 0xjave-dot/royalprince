Executive overview

What the website does

Royal Prince Fashion is designed as a mobile-first fashion shopping experience with a curated visual identity and a product-led journey. The storefront helps users discover products, browse by category, save favorites, manage a bag, apply vouchers, and complete a checkout flow that supports both personal purchases and gifting.

The site is not just a product listing page. It combines storefront UX, cart management, wishlist behavior, shared bag links, account settings, orders, shipping addresses, reviews, and a dedicated admin panel into one cohesive commerce system.

The current checkout flow is a polished demo flow. It behaves like a real handoff path and stores orders, but Paystack integration and final payment verification are not yet live.
Visual identity Premium mobile-first UI, curated daily accent color, bold typography, and soft glass-like surfaces.
Commerce flow Browse, wishlist, cart, checkout, sharing, and order confirmation.
Management layer Admin can manage catalog data, vouchers, new arrivals, reviews, and analytics.

Cart and checkout

Buying flow and payment handoff

The cart is built around a clean quantity-adjustment flow, with item removal, voucher application, subtotal and shipping math, and a visible order total. The checkout page then turns the bag into a more polished review experience.

  • Adjust item quantity directly in cart.
  • Remove items individually.
  • Apply and clear voucher codes.
  • See subtotal, discount, shipping, and total in real time.

Checkout supports a two-mode structure: "for myself" or "as a gift". Gift checkout includes recipient name and message fields. That gives the brand a more premium, more human purchase experience than a basic cart-to-payment handoff.

Shared bag links can be sent to another person so they can open the same basket and review exactly what they are paying for. The shared checkout preview includes product images, names, sizes, colors, quantities, and totals so the recipient understands the order before payment.

Guest checkout Users can continue without signing in and still complete the demo payment flow.
Payment preview The app simulates the payment handoff, so the client can see the intended end-to-end journey.
Order confirmation Successful checkout records an order and leads into a success screen with the order summary.

Account system

User profile and order data

Authentication

Firebase Auth handles sign-in and registration. The app can distinguish logged-in users from guests and adjusts the checkout flow accordingly.

Profile data

User account documents store settings, cart contents, wishlist items, shipping addresses, rewards, reviews, and order history.

Persistence model

The account layer is designed so the storefront state survives across sessions and can be merged with guest cart data when needed.

What the account layer enables

  • Saved cart, saved wishlist, and saved shipping addresses.
  • Order history for both registered and guest flows.
  • Reward summary tracking for future loyalty features.
  • Review history and profile settings such as size and currency preferences.

Admin panel

Control features for the business team

The admin panel is a full content and commerce control surface. It is not limited to a few settings. It gives the business team direct control over product data, category metadata, vouchers, curated new arrivals, reviews, and analytics.

Access is protected by authentication and an admin custom claim. The panel can therefore be presented as a secure operations dashboard for catalog and customer experience management.

Technical summary

How the system is built

Layer Implementation detail
Frontend React with Vite and TypeScript, styled using Tailwind CSS. The UI is componentized into layouts, shared cards, pages, and reusable utility modules.
Routing React Router powers the storefront, account, cart, checkout, and admin navigation paths.
Data storage Firebase Firestore stores account data, orders, reviews, catalog documents, vouchers, and analytics records.
Authentication Firebase Auth manages sign-in, registration, and access control to admin-only workflows.
Media Firebase Storage is used for catalog image uploads from the admin panel.
Share preview The project includes a server-rendered Open Graph preview path for shared product links, so external apps can render rich previews.
Responsive behavior Mobile-first layout with sticky top elements, bottom navigation, floating cart access, and safe-area aware spacing.

Technicalities worth mentioning in a pitch

  • The design system is consistent across storefront and admin surfaces, which gives the product a premium, intentional feel.
  • Cart and order data are normalized into reusable account documents, reducing inconsistencies across views.
  • The site already supports shared cart links and social preview metadata, which makes it easier to market products through messaging apps.
  • The admin panel is built for content operations, not just viewing data, so the client can run the store without needing a developer for every catalog change.
